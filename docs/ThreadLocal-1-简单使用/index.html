    ThreadLocal简单使用

    是秃子总会发光 ThreadLocal

    # 基本知识

    特性/类别 Thread ThreadLocal ThreadLocalMap
    职责 表示一个线程 提供线程局部变量 保存 ThreadLocal 变量的映射
    实例 每个线程有一个对应的 Thread 实例 可以有多个 ThreadLocal 实例，每个实例用于一个局部变量 每个 Thread 有一个 ThreadLocalMap 实例
    生命周期 与线程的生命周期相同 随着线程的生命周期，但 ThreadLocal 实例可以被回收 随着线程的生命周期，但可以被清理和回收
    存储数据 不直接存储线程局部变量 每个 Thread 中有一个 ThreadLocalMap，存储 ThreadLocal 变量 存储 ThreadLocal 变量的键值对
    存取方式 直接通过成员变量和方法存取线程相关的信息 通过 ThreadLocal 的 set、get、remove 方法存取 通过 ThreadLocalMap 的方法存取变量
    冲突问题 多个线程之间共享数据可能导致并发问题 每个线程独立拥有自己的 ThreadLocal 变量，不存在直接冲突 在多线程环境下可能存在冲突，需要处理同步问题

    # 使用场景

    # 单例类的线程安全问题

    • 场景： 解决多线程环境下单例类可能出现的线程安全问题。

    • 优势： 每个线程独立拥有自己的 ThreadLocal 变量，确保线程安全。

    # 数据库连接、Session管理

    • 场景： 在Web开发中，为每个请求创建数据库连接，确保在请求处理完毕后关闭连接。

    • 优势： 将连接与当前线程绑定，避免显式传递连接对象，提高代码简洁性。

    # 用户身份信息存储

    • 场景： 在Web应用中，共享用户身份信息（如用户ID）跨多个方法或类。

    • 优势： 将用户信息与当前线程绑定，避免显式传递信息，提高代码可读性和维护性。

    # 线程范围内的缓存

    • 场景： 需要为每个线程提供独立的缓存，以提高性能。

    • 优势： ThreadLocal 为每个线程提供独立的缓存，避免了显式同步，提高性能。

    # 线程安全的SimpleDateFormat

    • 场景： 避免多线程共享 SimpleDateFormat 实例可能导致的线程安全问题。

    • 优势： 使用 ThreadLocal 为每个线程创建独立的 SimpleDateFormat 实例，确保线程安全。

    # 线程池中的上下文传递

    • 场景： 在线程池处理任务时，需要在线程执行的过程中传递上下文信息。

    • 优势： 使用 ThreadLocal 在任务开始阶段设置上下文信息，避免显式传递信息。

    # 避免传递大量参数

    • 场景： 在复杂的业务场景中，需要传递大量参数给方法。

    • 优势： 使用 ThreadLocal 将参数与当前线程绑定，避免显式传递，提高代码清晰性。

    # 请求处理过程中的临时状态存储

    • 场景： 需要在请求处理的过程中存储临时状态信息。

    • 优势： 将临时状态信息存储到 ThreadLocal 中，避免显式传递，保持代码整洁。

    # 解决递归调用中的共享状态问题

    • 场景： 在递归调用中需要共享状态，但不希望被其他递归分支修改。

    • 优势： 使用 ThreadLocal 为每个递归分支提供独立的状态空间，避免状态混乱。

    # 全局共享对象的性能优化

    • 场景： 一些对象需要被多个线程频繁访问，但不需要同步。

    • 优势： 使用 ThreadLocal 为每个线程提供独立的对象实例，提高性能。

    # 简单使用

    public class ThreadLocalUtil<T> {
    private final ThreadLocal<T> threadLocal = new ThreadLocal<>();

    public void set(T value) {
        threadLocal.set(value);
    }

    public T get() {
        return threadLocal.get();
    }

    public void remove() {
        threadLocal.remove();
    }
}

    public class ThreadLocalExample {

    public static void main(String[] args) {
        // 创建一个ThreadLocalUtil实例，存储String类型的数据
        ThreadLocalUtil<String> threadLocalUtil = new ThreadLocalUtil<>();

        // 创建并启动两个线程
        Thread thread1 = new Thread(() -> {
            // 在线程1中设置数据到ThreadLocal
            threadLocalUtil.set("Data from Thread 1");
            // 模拟一些业务逻辑
            performBusinessLogic();
            // 获取并打印ThreadLocal中的数据
            System.out.println("Thread 1 Data: " + threadLocalUtil.get());
            // 在线程结束前清除ThreadLocal中的数据
            threadLocalUtil.remove();
        });

        Thread thread2 = new Thread(() -> {
            // 在线程2中设置不同的数据到ThreadLocal
            threadLocalUtil.set("Data from Thread 2");
            // 模拟一些业务逻辑
            performBusinessLogic();
            // 获取并打印ThreadLocal中的数据
            System.out.println("Thread 2 Data: " + threadLocalUtil.get());
            // 在线程结束前清除ThreadLocal中的数据
            threadLocalUtil.remove();
        });

        // 启动线程
        thread1.start();
        thread2.start();
    }

    private static void performBusinessLogic() {
        // 模拟业务逻辑的执行
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}
