    # Mysql体系结构

    组件 描述 示例
    连接池 用于管理客户端连接，通过复用连接提高系统的性能和伸缩性。 线程池，用于管理数据库连接线程
    SQL接口 接收和处理SQL查询的接口，提供与数据库的交互方式。 DDL，DML，Stored Procedures(存储过程)，Triggers（触发器）
    解析器 解析SQL查询，将其转换为服务器可以理解的内部格式，为后续的查询执行做准备。 SQL Parser（SQL解析器，用于解析SQL语句）
    查询优化器 对解析后的查询进行优化，找到最有效的执行方法。 Cost-based optimizer（基于成本的优化器，用于确定查询执行计划）
    缓存 存储查询结果，以便在接收到相同的查询时，可以直接从缓存中返回结果，提高查询效率。 Query Cache（查询缓存），InnoDB Buffer Pool（缓冲池，用于缓存数据和索引）
    系统管理和控制工具 提供一套完整的系统管理和控制工具，包括命令行工具、图形化管理工具、性能监控工具等，用于管理和监控MySQL服务器。 MySQL Workbench（图形化管理工具），Performance Schema（性能监控工具），mysqldump（用于备份数据库），mysqladmin（用于执行管理员操作）
    可插拔存储引擎 MySQL支持多种存储引擎，每种存储引擎都有其特定的特性和用途，可以根据需要选择合适的存储引擎。 InnoDB（支持事务处理和行级锁定），MyISAM（提供高速存储和检索，以及全文搜索能力），Memory（提供在内存中创建表的能力，用于快速访问）、Index（索引）
    系统文件 包括数据文件、日志文件、配置文件等，用于存储数据库的数据和元数据，记录数据库的操作日志，以及配置数据库的行为。 .frm（存储表定义的文件），.ibd（InnoDB存储引擎的数据文件），.MYD（MyISAM存储引擎的数据文件）等
    文件和日志 记录数据库的操作，包括查询日志、错误日志、慢查询日志、二进制日志等，用于监控和调试，以及在系统崩溃后恢复数据。 Error log（错误日志），Query log（查询日志），Slow query log（慢查询日志），Binary log（二进制日志），Redo log（重做日志，用于恢复事务）

    # 存储引擎概念

    ​ 存储引擎就是存储数据、建立索引、更新、查询数据等技术的实现方式。

    ​ 存储引擎是基于表的，而不是基于库的，所以存储引擎也可被称为表类型。

    • 可以在创建表时通过ENGINE指定存储引擎，默认是InnoDB
    • 可通过SHOW CREATE TABLE 表名查看表引擎
    CREATE TABLE `account` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `name` varchar(255) DEFAULT NULL,
  `balance` decimal(10,2) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB AUTO_INCREMENT=3 DEFAULT CHARSET=utf8mb4

    # 存储引擎分类

    可通过SHOW ENGINES查看当前数据库支持的存储引擎

    存储引擎 是否默认 描述 是否支持事务 是否支持行级锁定 是否支持外键
    InnoDB 是 支持事务处理，行级锁定，以及外键。 是 是 是
    MRG_MYISAM 否 一组相同的MyISAM表的集合，主要用于将多个MyISAM表作为一个表来查询。 否 否 否
    MEMORY 否 基于哈希的，存储在内存中，适用于临时表。 否 否 否
    BLACKHOLE 否 黑洞存储引擎，写入的任何数据都会消失，主要用于在主从复制中过滤掉不需要复制的数据。 否 否 否
    MyISAM 否 提供高速存储和检索，以及全文搜索能力，但不支持事务。 否 否 否
    CSV 否 CSV存储引擎，将数据以逗号分隔的格式存储，方便数据的导入导出。 否 否 否
    ARCHIVE 否 归档存储引擎，只支持INSERT和SELECT操作，适合存储和检索大量的历史数据。 否 否 否
    PERFORMANCE_SCHEMA 否 性能模式，用于收集数据库服务器性能参数，帮助调优。 否 否 否
    FEDERATED 否 联邦存储引擎，将数据存储在远程的MySQL数据库中，可以在本地服务器上创建链接到远程表的表。 否 否 否

    # InnoDB

    • 搞可靠，搞性能
    • MySql 5.5之后的默认引擎

    # 特点

    • DML操作遵循ACID模型、支持事务、行级锁、提高并发访问性能
    • 支持外键FOREIGN KEY约束，保证数据的完整性和正确性;

    # 文件

    • SHOW VARIABLES LIKE 'innodb_file_per_table'可以查看，参数是否为ON

    • 设置为ON时，每个InnoDB表都会使用一个独立的.ibd文件作为其表空间，存储该表的数据和索引，OFF则共用

    • XXX.ibd: innoDB引擎的每张表都会对应这样一个表空间文件，存储该表的表结构(frm、sdi)、数据和索引。
    • 参数： innodb_file_per_table

    • 默认ON，可以看见每张表都有自己的表空间
    • mysql8可以通过ibd2sdi xxx.ibd查看内容(JSON格式
    • )

    # 逻辑结构

    • 表空间：是最大的存储单位，可以包含多个段。
    • 段：是表空间中的一部分，每个段可以包含多个区。
    • 区：是段中的一部分，每个区包含多个页。
    • 页：是区中的一部分，每个页包含多个行。
    • 行：是页中的一部分，每个行存储一条记录的数据。

    每个行对应表中的一条记录，每个字段的数据都存储在行中

    • 区的大小是固定的，每个区包含1MB的空间。
    • 页的大小是固定的，每个页包含16KB的空间。

    # MyISAM

    早期版本默认的存储引擎

    # 特点

    • 不支持事务、外键、行锁
    • 支持表锁、访问速度快

    # 文件

    • xxx.sdi:存储表结构信息
    • XxxMYD:存储数据
    • xxx.MYI:存储索引

    # MEMORY

    # 特点

    • 表数据存储在内存，访问速度快，只适用于临时表和缓存
    • 支持hash索引

    # 文件

    xxx.sdi :存储表结构信息

    # 三种引擎对比总结

    特点 InnoDB MyISAM Memory
    存储限制 64TB 有 有
    事务安全 支持 不支持 不支持
    锁机制 行锁 表锁 表锁
    B+tree索引 支持 支持 支持
    Hash索引 支持 不支持 支持
    全文索引 支持(5.6版本之后) 支持 不支持
    空间使用 高 低 N/A
    内存使用 高 低 中等
    批量插入速度 低 高 高
    支持外键 支持 不支持 不支持

    # 存储引擎选择

    • 基本选择InnoDB（默认不用配置）
    • MyISAM往往使用MongoDB代替
    • MEMORY往往使用Redis代替
    存储引擎 选择条件
    InnoDB 1. 对事务的完整性有比较高的要求。 2. 在并发条件下要求数据的一致性。 3. 数据操作除了插入和查询之外，还包含很多的更新、删除操作。
    MyISAM 1. 应用是以读操作和插入操作为主。 2. 只有很少的更新和删除操作。 3. 对事务的完整性、并发性要求不是很高。
    MEMORY 1. 将所有数据保存在内存中，访问速度快，通常用于临时表及缓存。 2. 对表的大小有限制，太大的表无法缓存在内存中。 3. 无法保障数据的安全性。
