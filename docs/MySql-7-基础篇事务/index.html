    # 事务

    • 一组操作的集合，要么同时成功，要么同时失败。

    • mysql的事务时默认提交的

    # 语法

    提示

    • 不需要设置autocommit=0,使用默认的开启1即可
    • 只有提交后数据才会真正发生变化
    -- 开始事务(此时手动)
START TRANSACTION;

-- 查询账户1的余额
SELECT balance FROM account WHERE id = 1;

-- 查询账户2的余额
SELECT balance FROM account WHERE id = 2;

-- 转账操作
UPDATE account SET balance = balance - 100 WHERE id = 1;

-- 模拟错误，事务不会提交成功
SELECT book.b_id;

UPDATE account SET balance = balance + 100 WHERE id = 2;

-- 提交事务
COMMIT;

-- 出错时需要执行回滚
ROLLBACK;

    # 事务特性

    特性 英文名称 中文描述
    原子性 Atomicity 事务是一个原子操作，要么全部执行成功，要么全部不执行。如果在事务执行过程中发生错误，将回滚到事务开始前的状态。
    一致性 Consistency 事务使数据库从一个一致的状态转移到另一个一致的状态。在事务执行前后，数据库应该保持一致性，不管事务成功还是失败。
    隔离性 Isolation 多个事务可以并发执行，但是每个事务的执行应该与其他事务隔离，一个事务的中间状态对其他事务是不可见的，直到事务提交。
    持久性 Durability 一旦事务提交或回滚，其修改将永久保存在数据库中，即使系统崩溃或重启，事务的结果也应该得以保留。

    # 并发事务问题

    提示

    • 事务A虽然还没有提交，事务却可以读取到变更的数据，这与事务的隔离级别有关
    • 隔离级别定义了事务之间的可见性和影响
    问题 描述
    丢失更新 一个事务的更新可能被另一个事务的更新所覆盖，导致第一个事务的更改丢失。
    脏读 **一个事务读取了另一个事务尚未提交的数据。**如果另一个事务回滚，读取的数据就是无效的。
    幻读 一个事务按照条件查询数据时，没有对应的数据行，但是在插入数据时，又发现这行数据已经存在，好像出现了幻影。
    不可重复读 一个事务在多次读取同一行数据时，得到的结果不一致，可能是因为其他事务对该行数据进行了更新。
    死锁 两个或多个事务相互等待对方释放锁，导致它们都无法继续执行。
    阻塞 一个事务的操作被另一个事务的锁定操作所阻塞，导致性能下降。

    # 事务的隔离级别

    提示

    • 从上到下，安全性变高，性能变低
    • Mysql默认的系统隔离级别是可重复读

    # 隔离级别

    隔离级别 脏读 不可重复读 幻读
    读未提交（Read Uncommitted） 可能发生 可能发生 可能发生
    读提交（Read Committed） 避免 可能发生 可能发生
    可重复读（Repeatable Read） 避免 避免 可能发生
    串行化（Serializable） 避免 避免 避免
    -- 查看系统隔离级别
select @@global.tx_isolation;

-- 查看会话隔离级别(5.0以上版本)
select @@tx_isolation;

-- 查看会话隔离级别(8.0以上版本)
select @@transaction_isolation;
